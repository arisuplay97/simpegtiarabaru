Refactor project Next.js App Router + TypeScript + Tailwind ini menjadi demo SIMPEG PDAM enterprise yang matang, fokus pada role-based demo, sidebar yang realistis, user management, modul SP, dan akses yang tidak palsu.

WAJIB:
- Ganti seluruh istilah NIP menjadi NIK.
- Hapus seluruh istilah Eselon, ganti dengan Level Jabatan.
- Format golongan wajib model baru: A/I, A/II, B/I, B/II, C/I, C/II, D/I, D/II.
- Sidebar final hanya menampilkan modul yang benar-benar penting untuk demo.
- Mutasi, Dokumen Kepegawaian, Surat Keputusan, Lokasi Absensi, User Management, Role & Permission, dan SP / Surat Peringatan wajib ada.

SIDEBAR FINAL:
Dashboard: Dashboard Utama, Dashboard Direksi, Approval Center, Notifikasi
Kepegawaian: Data Pegawai, Profil Pegawai, Struktur Organisasi, Formasi Jabatan, Mutasi
Kehadiran: Absensi, Absensi Selfie, Cuti & Izin, Koreksi Absensi, Lokasi Absensi
Remunerasi: Payroll, Slip Gaji, Kenaikan Gaji Berkala
Kinerja & Karier: KPI & Penilaian, Kenaikan Pangkat
Administrasi: Dokumen Kepegawaian, Surat Keputusan, SP / Surat Peringatan
Pengaturan: User Management, Role & Permission

Hapus dari sidebar: Kontrak & Probation, Pensiun & Offboarding, Shift & Jadwal Kerja, Lembur, THR & Bonus, Potongan, BPJS, PPh 21, Disiplin & Penghargaan, Pelatihan, Sertifikasi, Laporan SDM, Laporan Absensi, Laporan Payroll.

LOGIN DEMO & ROLE:
- Tambahkan login demo role-based dengan role: Super Admin, HRD, Direktur, Pegawai.
- Simpan role aktif ke localStorage agar persist setelah refresh.
- Tampilkan role aktif di topbar atau sidebar.
- Tambahkan logout.

RESTRICTED ACCESS:
- Super Admin: semua akses.
- HRD: akses data pegawai, approval, mutasi, payroll, kgb, kenaikan pangkat, dokumen, SP, lokasi, KPI.
- Direktur: akses dashboard direksi, approval, data pegawai read-only, KPI, dokumen.
- Pegawai: hanya dashboard, profil sendiri, absensi, absensi selfie, cuti & izin, slip gaji, notifikasi, dokumen pribadi.
- Pegawai tidak boleh akses payroll global, user management, role & permission, mutasi global.
- Jika tidak punya akses: redirect ke /forbidden atau dashboard sesuai role.

APPROVAL FLOW:
- Implementasikan alur dummy bertahap:
  Draft -> Menunggu Review HRD -> Menunggu Persetujuan Direktur -> Disetujui / Ditolak.
- Terapkan minimal untuk Cuti, Mutasi, Kenaikan Pangkat, KGB, Koreksi Absensi.
- Role yang bisa approve:
  HRD hanya di tahap HRD.
  Direktur hanya di tahap Direktur.
  Pegawai tidak bisa approve.
- Tampilkan timeline approval sederhana.

USER MANAGEMENT:
- Tabel user: Nama, Email, NIK, Role, Status, Unit Kerja, Last Login.
- Aksi: Tambah User, Edit User, Ubah Role, Aktifkan/Nonaktifkan, Reset Password.
- Modal tambah/edit user wajib ada.
- User Management hanya untuk Super Admin dan HRD.

SP / SURAT PERINGATAN:
- Tabel SP: Nama, NIK, Jabatan, Unit, Status SP, Tanggal Terbit, Masa Berlaku, Keterangan, Aksi.
- Status SP: Tidak Ada SP, SP1, SP2, SP3.
- Aksi: Tambah SP, Edit SP, Cabut SP, Lihat Detail.
- HRD dan Super Admin bisa edit, Direktur lihat, Pegawai hanya bisa lihat dirinya jika diperlukan.

TOMBOL WAJIB HIDUP:
- Export Dashboard
- Generate Report
- Tambah Pegawai
- Proses Payroll
- Review / Setujui / Tolak di approval
- Download PDF / Print / Kirim Email slip gaji
Semua tombol minimal harus membuka modal, drawer, dialog, update state, atau toast. Tidak boleh CTA besar mati.
